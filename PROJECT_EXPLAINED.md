## 1. 项目是什么

### 1.1 一句话概括

**VisionFM 是一个眼科医生的"AI 助手"**，它能看眼底照片，自动画出血管的位置。

### 1.2 类比理解

想象你在教一个人学画画：

| 传统方式 | VisionFM 的方式 |
|----------|-----------------|
| 只教画猫（只能画猫） | 先学会画画的基本技巧（线条、阴影），再学画任何东西 |
| 需要 100 张猫的图才能学会 | 用 10 万张各种图就能学会基础技巧 |
| 换画狗要重新学 | 换画狗只需要练习一下 |

这就是"基础模型"和"普通模型"的区别。

### 1.3 VisionFM 能做什么

| 能力 | 说明 |
|------|------|
| **眼底血管分割** | 看眼底照片，把血管勾出来 |
| **疾病识别** | 看照片判断是否生病 |
| **多模态支持** | 能看眼底照片、OCT、B超等 8 种检查 |

### 1.4 项目结构

```
VisionFM 项目就像一个工厂：

├── 预训练车间（已完工）
│   └── 用 34 万张眼科图片训练出"基础能力"
│
├── 微调车间（可用）
│   └── 用少量专门数据让模型学会具体任务
│
└── 推理车间（我们用的）
    └── 拿训练好的模型去分析新图片
```

## 2. 底层原理

### 2.1 什么是"图像分割"

**图像分割**就是把图片中的不同部分分开：

```
原始照片 → 分割模型 → 标记结果

举例：眼底照片
         ↓
    [背景] [血管] [视盘]  ← 分割就是把每个像素归类
         ↓
    黑色像素  白色像素  灰色像素
```

### 2.2 什么是"深度学习"

**传统编程** vs **深度学习**：

```python
# 传统编程（你需要告诉计算机每一步规则）
def is_vessel(pixel):
    if pixel.color == red and pixel.brightness < 100:
        return True
    return False

问题：规则写不完，而且容易错
```

```python
# 深度学习（让计算机自己学规则）
model = train(看100万张图片, 这些图哪里是血管)

# 训练后，model 内部学会了"血管的特征"
result = model.predict(新图片)

# 就像人看多了图，直觉知道什么是血管
```

### 2.3 Vision Transformer (ViT) 原理

#### 传统 CNN 的做法

```
CNN 就像用放大镜一点一点看图：

[图片] → [滑动窗口] → [每次看一小块] → [合并结果]

优点：擅长看局部特征（边缘、纹理）
缺点：看全局关系能力弱
```

#### ViT 的做法

```
ViT 就像把图切成小方块，然后一起看：

[图片] → [切分成 196 个小块] → [同时看所有块] → [理解块之间的关系]

就像你看一幅画，不是只盯着一处，而是整体看，理解布局
```

**核心思想**：
1. 把 512×512 的图切成 16×16 的小方块（共 1024 个方块）
2. 每个小方块变成一串数字（"特征向量"）
3. 这些特征向量经过"注意力机制"处理，学会关注重要的部分
4. 最终输出分割结果

### 2.4 注意力机制 (Attention)

**"注意力"**就像你浏览图片时的眼睛：

```
图片：[一只猫在一个房间里]

传统 CNN：扫描每个位置，平等对待
Attention：发现"猫"很重要，给猫更多关注

权重分配：
[墙 0.1] [沙发 0.1] [猫 0.8] ← 注意力高
```

在 VisionFM 中，模型会自动学习：
- 血管的哪个部分重要
- 背景可以忽略
- 血管交叉怎么处理

### 2.5 UNETR 解码器

分割结果不是直接产生的，需要"解码器"：

```
ViT Encoder（理解图片）
    ↓
输出：1024 个特征向量，每个代表图的一小块
    ↓
UNETR Decoder（重建分割图）
    ↓
逐步放大：
    512×512 → 256×256 → 128×128 → 64×64
    同时融合不同层次的特征
    ↓
输出：最终的分割掩码
```

## 3. 技术栈

### 3.1 整体架构

```
用户上传图片
    ↓
[Python FastAPI] 接收请求
    ↓
[PyTorch] 加载模型
    ↓
[GPU/CPU] 执行推理
    ↓
返回结果
```

### 3.2 核心技术解释

| 技术 | 为什么用 | 作用 |
|------|----------|------|
| **Python** | 易读、生态丰富 | 整个项目的基础语言 |
| **PyTorch** | 深度学习主流框架 | 提供神经网络构建和训练功能 |
| **FastAPI** | 现代、高性能 Web 框架 | 让模型能通过网络被调用 |
| **Vision Transformer** | 处理图像效果好 | 提取图片特征 |
| **UNETR** | 专为分割设计 | 把特征转换成分割结果 |

### 3.3 为什么用 Transformer

传统 CNN（卷积神经网络）的问题：
- 看图范围有限（受卷积核大小限制）
- 难以理解全局（比如血管的整体走向）

Transformer 的优势：
- **注意力机制**：能学会关注重要部分
- **全局视野**：能理解整张图的关系
- **可扩展**：能处理不同大小的输入

## 4. 运行逻辑

### 4.1 整体流程

```
┌─────────────────────────────────────────────────────────────┐
│                   用户操作流程                          │
└─────────────────────────────────────────────────────────────┘

1. 用户上传眼底照片
    ↓
2. 图片传到后端服务器
    ↓
3. 后端加载预训练的模型
    ↓
4. 模型处理图片
   - 4.1 预处理：调整大小、颜色标准化
   - 4.2 特征提取：ViT 理解图片内容
   - 4.3 分割生成：UNETR 生成血管掩码
    ↓
5. 返回结果（原图 + 分割掩码）
    ↓
6. 前端叠加显示（用户看到血管被标出来）
```

### 4.2 详细步骤分解

#### 步骤 1：图片预处理

```python
# 原始图片可能各种大小、亮度
# → 统一处理成标准格式

1. 调整大小 → 512×512 像素
2. 转换颜色 → RGB 三通道
3. 亮度调整 → 如果太暗，自动增强（CLAHE）
4. 数值标准化 → 转成 0-1 的数字张量
5. 颜色归一化 → 减去均值，除以标准差

为什么要这样做？
→ 模型训练时就是这样学的，所以推理时也要一样
```

#### 步骤 2：特征提取（ViT Encoder）

```python
图片 → 切成 1024 个小块 → 每块转成特征向量
    ↓
经过 12 层 Transformer 处理
    ↓
输出：1024 个特征向量（每个代表一个位置的信息）

就像：
原始图：每个位置是像素颜色
处理后：每个位置变成了"这个位置有什么"的高级表示
```

#### 步骤 3：分割生成（UNETR Decoder）

```python
1024 个特征向量
    ↓
选择第 4、6、9、12 层的特征（多尺度信息）
    ↓
上采样 + 跳跃连接
    ↓
逐步恢复到 512×512
    ↓
输出：每个像素是血管的概率（0-1 之间的数字）
```

### 4.3 后处理

```python
概率图（0-1 的浮点数）
    ↓
阈值判断（比如 > 0.5 算血管）
    ↓
二值图像（0 或 255）
    ↓
返回给前端
```

### 5.2 核心代码解读

#### 代码 1：模型定义 (`models/vision_transformer.py`)

```python
class VisionTransformer(nn.Module):
    """
    Vision Transformer - 图片特征提取器

    可以理解成：一个"看图专家"
    """
    def __init__(self, ...):
        # 1. 初始化各个部件
        self.patch_embed = PatchEmbed()    # 切图工具
        self.pos_embed = PositionalEmbed() # 位置信息
        self.blocks = TransformerBlocks()  # 处理器（12 层）
        self.norm = LayerNorm()          # 标准化

    def forward(self, x):
        # x 是一张图片 [1, 3, 512, 512]

        # 步骤 1：切成小块
        x = self.patch_embed(x)
        # 结果：[1, 1024, 768]
        # 含义：1024 个位置，每个用 768 个数字描述

        # 步骤 2：加上位置信息
        x = x + self.pos_embed
        # 让模型知道"这个特征在图中的哪个位置"

        # 步骤 3：经过 12 层 Transformer
        for block in self.blocks:
            x = block(x)
        # 每层都会通过注意力机制提炼特征

        return x  # 输出提炼后的特征
```

**关键点**：
- `patch_embed`: 把图片从"像素"变成"特征"
- `blocks`: 不断提炼特征，像过筛子，越来越精炼
- `pos_embed`: 让模型知道特征的空间关系

#### 代码 2：分割头 (`models/unetr_head.py`)

```python
class Unetr_Head(nn.Module):
    """
    UNETR 解码器 - 把特征变成分割图

    可以理解成：一个"绘图专家"
    """
    def __init__(self, ...):
        # 初始化解码器部件
        self.d1 = DeconvBlock()  # 上采样
        self.s1 = ConvBlock()     # 特征融合
        self.c1 = ConvBlock()     # 卷积处理
        # ... 多层类似结构

    def forward(self, feats, inputs):
        # feats: 来自 ViT 的 4 层特征 [多尺度]
        # inputs: 原始图

        # 逐步上采样：
        # 从小尺寸 → 逐步放大 → 最终 512×512

        # 同时融合不同层次的信息（跳跃连接）
        x = self.d1(z12)      # 先放大最深层特征
        s = self.s1(z9)       # 处理较浅层特征
        x = torch.cat([x, s], dim=1)  # 拼接融合
        x = self.c1(x)        # 卷积提炼

        # ... 重复多次

        output = self.cls(x)  # 最终输出
        return output  # [1, 1, 512, 512] 每个像素是血管的概率
```

**关键点**：
- **多尺度融合**：用第 3/6/9/12 层特征，细节 + 全局信息
- **上采样**：逐步把小图放大回原尺寸
- **跳跃连接**：把浅层细节和深层语义融合

#### 代码 3：模型服务 (`model_service.py`)

```python
class SegmentationService:
    """
    模型服务 - 管理模型加载和推理
    """
    def __init__(self):
        self.device = 选择GPU或CPU
        self.model = None  # 开始是空的

    def load_model(self):
        # 1. 创建编码器（基础能力）
        encoder = models.vit_base(...)
        encoder.load_pretrained_weights(...)  # 加载预训练权重

        # 2. 创建解码器（专业知识）
        head = Unetr_Head(...)
        head.load_checkpoint(...)  # 加载专门训练的权重

        # 3. 组合
        self.model = SegModel(encoder, head)
        self.model.eval()  # 设置为预测模式

    def predict(self, image_bytes):
        # 1. 预处理图片
        img_tensor = self.preprocess(image_bytes)

        # 2. 推理
        with torch.no_grad():  # 不计算梯度（推理时不需要）
            output = self.model(img_tensor)
            prob = torch.sigmoid(output)  # 转成概率

        return prob.numpy()
```

**关键点**：
- `torch.no_grad()`: 推理不需要反向传播，关掉能省内存
- `eval()`: 关闭 dropout 等训练专用特性
- `sigmoid`: 把任意数字压到 0-1 之间

## 6. 为什么能运行

### 6.1 核心原理

**模型不是"编程"出来的，而是"训练"出来的。**

```
传统程序：
你要写清楚每条规则
if pixel是红色 and pixel在圆形区域:
    return "可能是血管"

深度学习：
你给模型看10万张图
每张图都有标注（哪里是血管）

模型自己学规律：
"哦，血管一般是红色的、细长的、有分支的..."
"背景一般是暗色的、均匀的..."

训练过程就是不断调整内部参数，直到能正确识别
```

### 6.2 为什么这样有效

| 人脑学习 | 深度学习 |
|----------|------------|
| 看多张图，形成经验 | 看多张图，形成参数 |
| 遇到新情况，用经验判断 | 遇到新图，用参数计算 |
| 经验多了就准确 | 数据多了就准确 |

**本质上都是"从经验中学习规律"**，只是实现方式不同。

### 6.3 预训练 + 微调的力量

```
【只用分割数据训练】
需要很多标注数据
只能学会分割这一件事

【预训练 + 微调】
预训练：先看 34 万张各种图，学会"眼科图像的基本规律"
微调：再用 1000 张分割标注图，学会"如何用基本规律做分割"

优势：
1. 需要的专门数据少（1000 张 vs 10000 张）
2. 效果更好（有基础知识打底）
3. 能泛化（没见过的例子也能处理）
```

### 6.4 真实世界类比

```
学医学生 vs 学专科医生

普通模型：
像从零开始培养医学生
需要学解剖、病理、诊断...

预训练模型：
像培养一个已经有基础的医学生
只要教他"看眼科照片"这个专科
他能把过去的背景知识用上
```

## 7. 关键概念总结

### 7.1 术语对照表

| 专业术语 | 通俗解释 |
|----------|----------|
| **推理 (Inference)** | 用训练好的模型预测新数据 |
| **分割 (Segmentation)** | 把图中的每个像素分类 |
| **掩码 (Mask)** | 标记结果图（黑白，白色是目标） |
| **注意力 (Attention)** | 模型学会"关注重要部分"的能力 |
| **编码器 (Encoder)** | 把图压缩成特征的过程 |
| **解码器 (Decoder)** | 把特征恢复成图的过程 |
| **微调 (Fine-tuning)** | 在预训练基础上，用少量数据专门训练 |
| **基础模型 (Foundation Model)** | 先在大规模数据上预训练，再适应各种任务 |

### 7.2 模型大小

```
checkpoint_108_linear.pth ≈ 200MB

这里面存的是什么？
→ 不是代码，而是"数字"

模型的参数大约有 1 亿个数字
每个数字 4 字节（float32）
所以文件约 400MB

加载后：
这 1 亿个数字被放入 GPU/CPU
推理时：
输入图片 → 数字运算 → 输出数字
```

### 7.3 为什么需要 GPU

```
单次推理的运算量：

输入：512×512×3 = 78 万个数字
中间计算：1024 个位置 × 768 维 × 12 层
输出：512×512 = 26 万个数字

CPU：每秒 10 亿次运算
GPU：每秒 1 万亿次运算

CPU 推理：可能需要 5-10 秒
GPU 推理：通常 0.1-0.5 秒
```

async def segment_image(file: UploadFile):

### 8.3 为什么返回 base64

```
方案对比：

【方案 1：返回文件路径】
问题：需要保存文件、管理存储、清理垃圾文件

【方案 2：直接返回二进制】
问题：HTTP 传输二进制需要额外处理

【方案 3：返回 base64】✓ 选择这个
优点：
1. 不需要服务器存储文件
2. 前端直接能显示 <img src="data:image/png;base64,...">
3. 无磁盘 I/O，速度快
4. 代码简单

缺点：
1. base64 比原图大约大 33%
2. 大图片可能超过请求大小限制

对于演示系统，方案 3 最合适
```

### 9.2 实践理解方式

```python
# 看代码时，思考每个变量的物理意义

input_tensor.shape  # torch.Size([1, 3, 512, 512])
# → 这是：1 张图，3 个颜色通道，512×512 大小

output.shape  # torch.Size([1, 1, 512, 512])
# → 这是：1 张图，1 个通道（黑白），512×512 大小

prob > 0.5
# → 这是：概率超过 50%，算作血管
```

## 10. 总结

### 10.1 一句话总结

**VisionFM 是一个通过学习 34 万张眼科图像，掌握了眼科图像特征，然后专门训练血管分割能力的 AI 模型。**

### 10.2 核心流程图

```
【数据】34 万张眼科图片 → 【预训练】基础模型
                                  ↓
                        【+】1000 张分割标注 → 【微调】分割头
                                  ↓
                            【组合模型】
                                  ↓
【用户】上传新图 → 【推理】得到血管分割 → 【返回】结果
```

### 10.3 关键技术点

1. **Vision Transformer**：把图片切成小块，用注意力机制理解
2. **UNETR**：把特征解码回分割图
3. **预训练 + 微调**：先学基础，再学专业
4. **FastAPI**：让模型能被 Web 调用
