# VisionFM 分类模型性能改进指南

## 问题分析

### 当前性能问题
- **F1分数低**：最高仅0.3780（37.8%）
- **准确率波动**：在48.5%左右，不稳定
- **过拟合明显**：训练损失低但验证性能差
- **各类别性能不均**：少数类识别率极低

### 根本原因：类别不平衡
- **类别0**：约36%（多数类）
- **类别1和4**：各约8-10%（少数类）
- **类别2和3**：各约20-24%（中等）
- **类别4**：约5-10%（极少数类）

## 改进方案实施

### 1. 类别权重平衡（已实施）

**原理**：为少数类分配更高权重，使模型更关注难分类样本

**实现**：
```python
# 计算类别权重
class_weights = compute_class_weight('balanced', class_counts)
# 加权损失函数
loss = nn.CrossEntropyLoss(weight=class_weights)(output, target)
```

**效果预期**：提升少数类召回率20-30%

### 2. 平衡采样（已实施）

**原理**：每个批次中保持类别平衡，避免多数类主导训练

**实现**：
```python
# WeightedRandomSampler
sample_weights = [1.0 / class_counts[target] for target in targets]
sampler = WeightedRandomSampler(sample_weights, replacement=True)
```

**效果预期**：提升模型对各类别的学习能力

### 3. 增强数据增强（已实施）

**原理**：对少数类应用更强的数据增强，增加样本多样性

**实现**：
- 少数类（类别4）：旋转25°、颜色抖动0.4、随机擦除0.4
- 多数类：标准增强

**效果预期**：提升模型泛化能力，减少过拟合

### 4. 焦点损失（已实施）

**原理**：对难分类样本给予更高权重，关注分类边界样本

**实现**：
```python
class FocalLoss(nn.Module):
    def forward(self, inputs, targets):
        ce_loss = F.cross_entropy(inputs, targets, reduction='none')
        pt = torch.exp(-ce_loss)
        focal_loss = self.alpha * (1-pt)**self.gamma * ce_loss
        return focal_loss.mean()
```

**效果预期**：提升难分类样本识别率15-25%

### 5. 改进分类器架构（已实施）

**原理**：增加正则化，防止过拟合，提升模型稳定性

**实现**：
- 批归一化层
- 层归一化
- Dropout(0.3)
- 标签平滑(0.1)

**效果预期**：提升泛化能力，减少过拟合

### 6. 学习率调度优化（已实施）

**原理**：动态调整学习率，提升收敛速度和稳定性

**实现**：
```python
# OneCycleLR调度器
scheduler = torch.optim.lr_scheduler.OneCycleLR(
    optimizer, max_lr=args.lr, epochs=args.epochs, 
    steps_per_epoch=len(train_loader)
)
```

**效果预期**：更快收敛，避免局部最优

### 参数调优建议

#### 学习率
- 基础：0.001
- 高级：0.0005-0.002（根据验证效果调整）

#### 批大小
- 小数据集：16-32
- 大数据集：64-128

#### 正则化强度
- Dropout：0.2-0.5
- 权重衰减：0.01-0.1

#### 训练轮数
- 观察验证指标，早停防止过拟合
- 通常50-100轮足够

## 预期效果

### 性能提升目标
- **F1分数**：从0.378提升到0.45-0.55
- **准确率**：从48.5%提升到55-65%
- **少数类召回率**：从<10%提升到30-40%
- **各类别平衡性**：显著改善

### 训练监控指标
1. **观察损失曲线**：验证损失应与训练损失同步下降
2. **监控各类别指标**：确保少数类性能持续改善
3. **学习率变化**：OneCycleLR应有周期性变化
4. **验证集稳定性**：避免指标剧烈波动

## 故障排除

### 如果过拟合仍然严重
1. 增加Dropout到0.5
2. 增加数据增强强度
3. 减少模型参数（减少层数）
4. 使用更强的正则化（L2权重衰减）

### 如果收敛缓慢
1. 增加初始学习率
2. 减少批大小
3. 检查数据预处理
4. 尝试不同的优化器（AdamW、SGD）

### 如果少数类性能仍差
1. 手动调整类别权重
2. 尝试过采样而非欠采样
3. 使用更复杂的损失函数（如Label Smoothing）
4. 考虑两阶段训练策略
