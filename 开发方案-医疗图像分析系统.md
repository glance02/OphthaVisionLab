# 医疗图像分析系统开发方案

## 一、项目概述

开发一个Web应用，用户上传眼底（Fundus）照片后，系统自动完成：
1. **DR（糖尿病视网膜病变）二分类预测** - 判断是否患病
2. **病变区域分割** - 标记出血管和病变区域
3. **AI分析报告生成** - 使用通义千问API生成专业分析报告

**项目背景**：大创项目，需要完整的软件系统用于答辩演示

## 三、技术方案

### 选择：**Streamlit**

**选择理由：**
- 纯Python开发，不需要HTML/CSS/JavaScript
- 开发速度快，1-2天可完成基本功能
- 界面自动生成，适合答辩演示
- 文档丰富，社区活跃

### 第二步：创建Streamlit应用

**新建文件：`dr_analysis_app.py`**

**界面布局：**
```
┌─────────────────────────────────────┐
│     眼底图像分析系统                 │
├─────────────────────────────────────┤
│ [上传图片按钮]                       │
│                                     │
│ 显示上传的原图                       │
│                                     │
│ [开始分析] 按钮                      │
│                                     │
├─────────────────────────────────────┤
│ DR预测结果 │ 分割结果               │
│ - 类别: DR阳性  │ [分割图像]        │
│ - 概率: 85.6%  │                    │
│ - 置信度: 高    │                    │
├─────────────────────────────────────┤
│ AI分析报告（Markdown格式）          │
│ ...详细分析内容...                  │
└─────────────────────────────────────┘
```

**需要实现的功能：**
1. 文件上传：`st.file_uploader`
2. 图片显示：`st.image`
3. 分析按钮：`st.button`
4. 结果展示：使用 `st.columns` 实现两列布局
5. 报告展示：`st.markdown`

---

### 第三步：集成大模型API

**使用：通义千问API**

**需要做的：**
1. 注册阿里云账号，开通通义千问服务
2. 获取API Key
3. 编写函数调用API
4. 设计Prompt模板

**Prompt设计思路：**
```
你是眼科医生助理，根据以下AI分析结果生成报告：

DR预测：{类别}，概率{xx}%，置信度{高/中/低}
分割结果：病变面积{xx}px²，占比{xx}%

请生成包含以下内容的报告：
1. 风险评估
2. 主要发现
3. 建议措施
4. 随访建议
```
