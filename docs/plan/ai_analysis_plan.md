# VisionFM AI 分析系统集成方案

> 方案类型：API 调用多模态大模型
> 创建日期：2025-01-10
> 状态：待实施

---

## 一、方案概述

### 核心思路

将现有的二分类模型和分割模型作为"前端特征提取器"，多模态大语言模型作为"后端分析大脑"。

```
输入图像 → 本地模型推理 → 提取结构化结果 → 发送给API → AI生成自然语言分析
```

### 为什么选择这个方案？

| 优势 | 说明 |
|------|------|
| 开发量最小 | 3-4天可完成基础版本 |
| AI理解能力强 | 多模态LLM已见过大量医学图像 |
| 无需微调 | 通过Prompt工程即可达到良好效果 |
| 无需本地GPU | 推理在云端完成 |
| 灵活可调 | 可随时切换更好的模型或调整Prompt |

## 三、技术选型

### API 服务：SiliconFlow

| 特性 | 说明 |
|------|------|
| 访问速度 | 国内访问快，不需要翻墙 |
| 模型选择 | 支持Qwen2-VL、InternVL等多种视觉模型 |
| 成本 | 比OpenAI便宜很多 |
| API格式 | 与OpenAI兼容，易于集成 |

### 推荐模型

| 模型 | 特点 | 推荐场景 |
|------|------|----------|
| Qwen2-VL | 阿里通义千问视觉版，中文理解好 | 首选 |
| InternVL | 书生浦项，医学影像表现不错 | 可备选 |
| LLaVA | 开源经典视觉语言模型 | 可测试 |

## 五、核心功能设计

### 5.1 发送给API的内容

**图片部分：**
- 原始眼底照片
- 分割掩码叠加图（让AI看到识别区域）
- 可选：热力图、对比图

**文字数据：**
```json
{
  "classification": {
    "probability": 0.87,
    "class": "疾病"
  },
  "segmentation": {
    "vessel_area_ratio": 0.15,
    "vessel_density": 0.42,
    "tortuosity_index": 1.23
  }
}
```

### 5.2 Prompt 模板设计

```
你是一位专业的眼科影像分析助手。

【任务】
请分析这张眼底照片，并结合模型检测结果给出专业意见。

【模型检测结果】
- 疾病概率：{probability}%
- 血管面积比：{vessel_area_ratio}
- 血管密度：{vessel_density}
- 血管弯曲度：{tortuosity_index}

【输出要求】
1. 影像所见描述
2. 异常发现分析
3. 风险评估
4. 进一步检查建议

请用专业但易懂的语言回答，供医生参考。
```

### 5.3 输出报告格式

```json
{
  "patient_id": "P001",
  "image_id": "IMG_20250110_001",
  "timestamp": "2025-01-10T10:30:00",

  "model_results": {
    "classification": {
      "probability": 0.87,
      "class": "疾病"
    },
    "segmentation": {
      "mask_path": "./output/masks/IMG_001_mask.png",
      "vessel_area_ratio": 0.15,
      "vessel_density": 0.42
    }
  },

  "ai_analysis": {
    "findings": "视网膜血管密度异常增高...",
    "abnormalities": "血管弯曲度超出正常范围...",
    "risk_level": "高风险",
    "recommendation": "建议立即进行进一步眼科检查"
  },

  "visualization": "./output/overlay/IMG_001_overlay.png"
}
```

## 七、成本估算

### SiliconFlow 定价（参考）

| 模型 | 输入价格 | 输出价格 |
|------|----------|----------|
| Qwen2-VL | ¥0.003/1K tokens | ¥0.006/1K tokens |
| InternVL | ¥0.005/1K tokens | ¥0.008/1K tokens |

### 单次分析成本估算

```
假设：
- 图片编码后约 1000 tokens
- 输入文字约 500 tokens
- AI输出约 1000 tokens

单次成本 ≈ (1000+500) × 0.003 + 1000 × 0.006 ≈ ¥0.01-0.02
```

### 月度成本估算

| 使用量 | 成本 |
|--------|------|
| 100次/月 | ¥1-2 |
| 1000次/月 | ¥10-20 |
| 10000次/月 | ¥100-200 |

## 八、注意事项

### 8.1 数据隐私

- ⚠️ 患者图像和数据会发送到第三方API
- 建议对患者信息进行脱敏处理
- 如果是医院使用，需确认数据合规要求

### 8.2 API限制

- 有并发请求限制
- 有速率限制
- 建议实现重试机制

### 8.3 责任声明

- AI分析结果仅供参考，不能替代医生诊断
- 在使用界面应加入免责声明

## 九、后续优化方向

1. **增加本地缓存**：相同图像不重复调用API
2. **批量处理优化**：异步处理提升吞吐量
3. **多模型对比**：调用多个模型，综合分析
4. **反馈学习**：记录医生修改意见，优化Prompt
5. **切换本地模型**：如需要隐私保护，可部署本地视觉语言模型
